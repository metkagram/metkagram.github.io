# Metkagram partnership brief

## Thesis

Metkagram is an open language-learning and language-data system built around a single structured asset: a versioned corpus that links annotated sentences, reusable B2–C1 patterns, study sets, translations and provenance.

The same source data supports two product surfaces:

- a readable learning interface for people;
- a static API and dataset for agents, researchers and language tooling.

This creates compounding value without making the public website dependent on an expensive runtime backend.

## Verified project assets

Project counts are generated from the validated source data at build time and displayed on `/en/support/`.

- more than 1,000 complete English/German reusable patterns;
- 25,116 annotated sentences;
- 2,240 curated document sets;
- 80 named study sets;
- a canonical annotation schema and migration report;
- a static API, search index, OpenAPI description and read-only MCP bridge;
- original Android and iOS applications.

These are content and infrastructure metrics, not audience, revenue or efficacy metrics.

## Next milestone packages

### 1. Method validation

Deliverables: preregistration, reviewed stimuli, pilot execution, analysis code and public report.

Success measure: a completed and reproducible comparison study, regardless of whether the result is positive, null or negative.

### 2. Editorial quality

Deliverables: annotation sample design, independent review, adjudication workflow and regression fixtures.

Success measure: label-level agreement and a documented correction rate across English and German collections.

### 3. Localization pipeline

Deliverables: schema extension, coverage report, translator workflow, English fallback rules and one new target-language pilot.

Success measure: a complete sample collection that passes validation without copied UI logic.

### 4. Learner retention

Deliverables: personal learning path, active-retrieval loop, privacy-preserving event model and opt-in outcome instrumentation.

Success measure: a defined cohort can move from reading to delayed retrieval with measurable, auditable events.

## Partnership models

- Mission-aligned investment tied to a milestone package.
- Research partnership with preregistration and publication independence.
- Linguistic or editorial contribution with explicit provenance.
- Technology partnership for accessibility, data quality, distribution or infrastructure.
- Commercial licensing that does not remove the free core or rewrite research conclusions.

## Integrity rules

- Do not imply an active funding round unless its terms and owner are defined.
- Do not publish user, revenue or learning-outcome numbers without auditable evidence.
- Sponsors cannot alter annotations, research findings or recommendations for their own interests.
- Disclose material support and preserve record-level provenance.
- Keep the core learning materials publicly accessible.

The public partnership page is generated at `/en/support/`.
